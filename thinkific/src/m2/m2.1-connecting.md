# Connecting an Assistant to a Deployment

In this lesson, you'll learn how to connect a LangSmith **assistant** to a deployed LangGraph agent — and run your first conversation thread against it.

An assistant is a configuration layer on top of a deployed graph. It lets you:
- Set a custom system prompt
- Configure model and tool settings
- Create multiple named variants of the same graph

By the end of this lesson you will be able to:
1. Create a LangGraph client pointing at a deployment
2. List available graphs
3. Create and configure an assistant
4. Run a conversation thread
5. Clean up by deleting the assistant

The steps below walk through `python/l2/2.1-connecting.py` (and its TypeScript equivalent). Open the file alongside this lesson — the code is the lesson. To run it:

## Step 1: Connect to Your Deployment

Create a client pointing at your deployment URL using your LangSmith API key.

**Finding your URL:**
- **Local (LangGraph Studio):** `http://127.0.0.1:2024`
- **LangSmith Deployment:** find the URL on your deployment's page in the LangSmith UI under *Deployments*

The API key is read from your environment — it should never be hardcoded.

<!-- block: l2.1-step1-code -->
<Tabs>
  <Tab title="Python">
```python
DEPLOYMENT_URL = os.environ.get("LANGGRAPH_URL", "http://127.0.0.1:2024")
API_KEY = os.environ.get("LANGSMITH_API_KEY", "")

client = get_client(url=DEPLOYMENT_URL, api_key=API_KEY)
```
  </Tab>


## Step 4: Set the Assistant Context

The agent reads `lesson_id` and `student_name` from the assistant `context` at runtime — it uses `lesson_id` to load the correct lesson instructions and material, and `student_name` to personalize responses and look up student data from the store.

Context is stored server-side on the assistant record. You do NOT resend it on every call — the server applies it automatically whenever you use that `assistant_id`.

> **Important:** When updating, include ALL context fields, not just the ones you are changing. The update call replaces the entire context object.

<!-- block: l2.1-step4-code -->
<Tabs>
  <Tab title="Python">
```python
assistant = await client.assistants.update(
    assistant["assistant_id"],
    context={
        "lesson_id": "tutor_l1",
        "student_name": "Student",
    },
)
print(f"Updated assistant: {assistant['assistant_id']}")
```
  </Tab>
  <Tab title="TypeScript">
```typescript
assistant = await client.assistants.update(assistant.assistant_id, {
```

## Step 5: Run a Thread Using the Assistant

A **thread** holds the conversation history. Create a thread, then run it against your assistant using `runs.wait()` — this waits for the full response and returns the final state.

Two key concepts:
- **Context** (`lesson_id`, `student_name`) is stored on the assistant — you set it once.
- **Config** (e.g., `thread_id`) is passed per run — it identifies the conversation.

<!-- block: l2.1-step5-code -->
<Tabs>
  <Tab title="Python">
```python
thread = await client.threads.create()
print(f"\nThread: {thread['thread_id']}")

result = await client.runs.wait(
    thread["thread_id"],
    assistant_l1["assistant_id"],
    input={"messages": [{"role": "user", "content": "What lesson number is this?"}]},
)
messages = result.get("messages", [])
if messages:
    content = messages[-1].get("content", "")
    if isinstance(content, list):
        content = "".join(
            b.get("text", "") for b in content
            if isinstance(b, dict) and b.get("type") == "text"
        )
    print(content)
```
  </Tab>
  <Tab title="TypeScript">
```typescript
const thread = await client.threads.create();
console.log(`\nThread: ${thread.thread_id}`);

const result = await client.runs.wait(thread.thread_id, assistant.assistant_id, {
  input: { messages: [{ role: "user", content: "What lesson number is this?" }] },
});
const messages = (result as Record<string, unknown[]>).messages ?? [];
if (messages.length > 0) {
  const last = messages[messages.length - 1] as Record<string, unknown>;
  let content = last.content;
  if (Array.isArray(content)) {
    content = (content as Array<unknown>)
      .filter((b): b is Record<string, unknown> => typeof b === "object" && b !== null && (b as Record<string, unknown>).type === "text")
      .map((b) => (b as Record<string, unknown>).text ?? "")
      .join("");
  }
  console.log(content ?? "");
}
```

## Step 5b: A Second Assistant — Same Question, Different Context

Now create `assistant_l2` pointing at lesson 2. Ask the same question — the agent will answer based on the lesson 2 context, demonstrating that context is what makes each assistant behave differently. This time we stream the response.

<!-- block: l2.1-step5b-code -->
<Tabs>
  <Tab title="Python">
```python
assistant_l2 = await client.assistants.create(
    graph_id="tutor",
    name="Tutor — Lesson 2",
    context={
        "lesson_id": "tutor_l2",
        "student_name": "Student",
    },
)

thread_l2 = await client.threads.create()

printed: dict[str, int] = {}
async for event in client.runs.stream(
    thread_l2["thread_id"],
    assistant_l2["assistant_id"],
    input={"messages": [{"role": "user", "content": "What lesson number is this?"}]},
    stream_mode="messages",
):
    if not isinstance(event.data, list):
        continue
    for item in event.data:
        msg = item[0] if isinstance(item, (list, tuple)) else item
        if not isinstance(msg, dict):
            continue
        if msg.get("type") in ("AIMessageChunk", "AIMessage", "ai"):
            msg_id = msg.get("id", "")
            content = msg.get("content", "")
            if isinstance(content, str):
                already = printed.get(msg_id, 0)
                if len(content) > already:
                    print(content[already:], end="", flush=True)
                    printed[msg_id] = len(content)
print()
```
  </Tab>
  <Tab title="TypeScript">
```typescript
const assistantL2 = await client.assistants.create({
  graphId: "tutor",
  name: "Tutor — Lesson 2",
  context: {
    lesson_id: "tutor_l2",
    student_name: "Student",
  },
});

const threadL2 = await client.threads.create();

const printed: Record<string, number> = {};
const stream = client.runs.stream(threadL2.thread_id, assistantL2.assistant_id, {
  input: { messages: [{ role: "user", content: "What lesson number is this?" }] },
  streamMode: "messages",
});
for await (const event of stream) {
  if (!Array.isArray(event.data)) continue;
  for (const item of event.data as Array<unknown>) {
    const msg = Array.isArray(item) ? item[0] : item;
    if (!msg || typeof msg !== "object") continue;
    const m = msg as Record<string, unknown>;
    if (m.type === "AIMessageChunk" || m.type === "AIMessage" || m.type === "ai") {
      const msgId = (m.id as string) ?? "";
      const content = typeof m.content === "string" ? m.content : "";
      const already = printed[msgId] ?? 0;
      if (content.length > already) {
        process.stdout.write(content.slice(already));
        printed[msgId] = content.length;
      }
    }
  }
}
console.log();
```
  </Tab>
</Tabs>

## Recap

- **Assistants** are configuration records layered on top of a deployed graph — they hold `lesson_id`, `student_name`, and any other per-assistant settings.
- **Context** (`context` field) is stored server-side on the assistant — set it once, applied automatically on every run.
- **Config** (e.g., `thread_id`) is ephemeral — passed per run to identify the conversation thread.
- Two assistants on the same graph behave differently when given different context — no code change required.
- Deleting an assistant frees its config record only — it does not affect compute.
