# The Tutor UI

In this lesson you'll start up the tutor chat UI and walk through the key pieces of code that connect it to your deployed agent. The UI uses a server proxy pattern — a local web server uses the LangGraph SDK to create a client, manage threads, and stream responses from your deployment. The browser talks to the local server; the local server talks to LangGraph.

---

## Architecture

```
Browser (index.html)
      |
      | POST /chat
      v
Local Server (server.py / server.ts)
      |
      | LangGraph SDK
      v
LangGraph Deployment (agent.py / agent.ts)
      |
      | LLM provider
      v
LLM
```

The browser sends messages to the local server, which initializes a LangGraph client and forwards requests to your deployment.

## The Config Tab

The **config** tab has two sections:

- **Deployment URL** — where your LangGraph agent is running. Defaults to `http://127.0.0.1:2024` for local Studio.
- **Student profile** — your name, goals, and learning preferences. Click **start** to create a session, which sets up one assistant and thread per lesson.

The deployment URL is passed to the server on every request — this lets you switch between local and deployed agents without restarting the server.

## Key Code: The Server Proxy

The server (`server.py` / `server.ts`) has two jobs:

1. Serve `index.html`
2. Proxy `/chat` requests to the LangGraph deployment

The browser passes `assistant_id` and `thread_id` directly — both are created at session start and stored in the browser's localStorage.

    client = get_client(req.deployment_url)
    return StreamingResponse(
        stream_response(client, req.thread_id, req.assistant_id, req.message),
        media_type="text/plain",

  const { message, deployment_url, assistant_id, thread_id } = req.body;
  const client = getClient(deployment_url);
  res.setHeader("Content-Type", "text/plain");

### Create Student Sessions

Called once when the student clicks **start**. Creates one assistant and one thread per lesson — each assistant is configured with the student's context so it teaches the right lesson.

<!-- block: l2.2-create-sessions -->
<Tabs>
  <Tab title="Python">
```python
async def create_student_sessions(
    client: LangGraphClient,
    namespace: str,
    student_name: str = "Student",
) -> list[dict]:
    sessions = []
    for lesson in LESSONS:
        assistant = await client.assistants.create(
            graph_id="tutor",
            name=f"{namespace} — {lesson['id']}",
            context={
                "lesson_id": lesson["id"],
                "store_namespace": namespace,
                "student_name": student_name,
            },
        )
        thread = await client.threads.create()
        sessions.append({
            "lesson_id": lesson["id"],
            "assistant_id": assistant["assistant_id"],
            "thread_id": thread["thread_id"],
        })
    return sessions
```

The `assistant_id` and `thread_id` for each lesson are saved to localStorage — the browser sends them on every `/chat` request so the server always routes to the right assistant and conversation thread.

### Stream a Response

`stream_mode="messages"` sends events as the agent runs. Each event may contain the same message multiple times as state updates accumulate, so we track cumulative content per message ID and only yield new characters.

Some LLMs (e.g. Gemini 2.5 Flash) return content as a list of blocks rather than a plain string, so we normalize that too.

<!-- block: l2.2-stream -->
<Tabs>
  <Tab title="Python">
```python
async def stream_response(
    client: LangGraphClient,
    thread_id: str,
    assistant_id: str,
    message: str,
) -> AsyncGenerator[str, None]:
    printed: dict[str, int] = {}  # track cumulative content per message ID
    async for event in client.runs.stream(
        thread_id,
        assistant_id,
        input={"messages": [{"role": "user", "content": message}]},
        stream_mode="messages",
    ):
        if not isinstance(event.data, list):
            continue
        for item in event.data:
            msg = item[0] if isinstance(item, (list, tuple)) else item
            if not isinstance(msg, dict):
                continue
            if msg.get("type") in ("AIMessageChunk", "AIMessage", "ai"):
                msg_id = msg.get("id", "")
                content = msg.get("content", "")
                if isinstance(content, list):
                    parts = []
                    for block in content:
                        if isinstance(block, str):
                            parts.append(block)
                        elif isinstance(block, dict):
                            if block.get("type") == "text" and not block.get("extras", {}).get("signature"):
                                parts.append(block.get("text", ""))
                    content = "".join(parts)
                if isinstance(content, str):
                    already = printed.get(msg_id, 0)
                    if len(content) > already:
                        yield content[already:]
                        printed[msg_id] = len(content)
```
  </Tab>
  <Tab title="TypeScript">
```typescript
async function* streamResponse(
  client: Client,
  threadId: string,
  assistantId: string,
  message: string,
): AsyncGenerator<string> {
  const printed: Record<string, number> = {};
  const stream = client.runs.stream(threadId, assistantId, {
    input: { messages: [{ role: "user", content: message }] },
    streamMode: "messages",
  });
  for await (const event of stream) {
    if (!Array.isArray(event.data)) continue;
    for (const item of event.data) {
      const msg = Array.isArray(item) ? item[0] : item;
      if (!msg || typeof msg !== "object") continue;
      const m = msg as Record<string, unknown>;
      if (m.type === "AIMessageChunk" || m.type === "AIMessage" || m.type === "ai") {
        const msgId = (m.id as string) ?? "";
        let content = m.content ?? "";
        if (Array.isArray(content)) {
          content = content
            .map((b: unknown) => (typeof b === "string" ? b : (b as Record<string, unknown>).text ?? ""))
            .join("");
        }
        if (typeof content === "string") {
          const already = printed[msgId] ?? 0;
          if (content.length > already) {
            yield content.slice(already);
            printed[msgId] = content.length;
          }
        }
      }
    }
  }
}
```

## Recap

- The UI uses a **server proxy pattern** — the browser talks to a local server, which uses the LangGraph SDK to connect to your deployment.
- The **deployment URL** is entered in the config tab — it's passed to the server on each request, making it easy to switch between local and deployed agents.
- Clicking **start** calls `create_student_sessions` — one assistant and one thread per lesson, each assistant configured with the student's context.
- The `assistant_id` and `thread_id` are stored in localStorage and sent on every `/chat` request — no session state on the server.
- `stream_response` tracks cumulative content per message ID to handle LangGraph's repeated state update events, and normalizes list-format content blocks for LLMs that return them (e.g. Gemini 2.5 Flash).
