In this exercise you will deploy the `dacli_tutor` project — a deepagents CLI agent that uses the same tutor skills you have been working with. By the end you will have a running tutor accessible via a built-in web UI.

The project is in `python/m6/dacli_tutor/` in the course repository.

---

## Step 1 — Set Up Your .env

The deepagents CLI bundler expects a `.env` file alongside `deepagents.toml` at the project root. It does NOT walk up the directory tree to find one — so even though the course has a shared `.env` at `python/.env`, the CLI needs a separate `.env` inside the project. Copy the template and fill in your keys:

```bash
cd lca-deployment/python/m6/dacli_tutor
cp .env.example .env
```

Edit `.env` and set:

```bash
LANGSMITH_API_KEY=lsv2_...
LANGSMITH_PROJECT=lca-deployment
LANGSMITH_TRACING=true
ANTHROPIC_API_KEY=your-anthropic-api-key
```

`dacli_tutor` uses `anthropic:claude-sonnet-4-6` by default. If you prefer a different model, update both the corresponding key in `.env` and the `model` field in `deepagents.toml`.

---

## Step 2 — Explore the Project

Before running anything, look at the files in `dacli_tutor/`:

**`deepagents.toml`** — the config

```toml
[agent]
name = "deep_tutor"
model = "anthropic:claude-sonnet-4-6"

[auth]
provider = "anonymous"

[frontend]
enabled = true
app_name = "LangChain Tutor"

[memories]
backend = "store"
```

A name, a model, anonymous auth, a frontend UI, and `[memories]` configured to use the deployment's Store as the backing layer for `AGENTS.md` and `skills/`. No sandbox, no subagents.

**`AGENTS.md`** — the system prompt

The tutor's instructions: how to teach, how to load skills, what to do when a student asks a question outside the current module.

**`mcp.json`** — the LangChain docs tool

```json
{
  "mcpServers": {
    "docs": {
      "url": "https://docs.langchain.com/mcp",
      "transport": "http"
    }
  }
}
```

One HTTP MCP server. The agent can query LangChain documentation directly.

**`skills/module-1/`** contains two files:

- `SKILL.md` — teaching instructions for the deployment architecture module
- `information.md` — reference material the agent reads when answering factual questions

---

## Step 3 — Run Locally

Navigate to the project directory and start the dev server:

```bash
cd lca-deployment/python/m6/dacli_tutor
uv run deepagents dev
```

The server starts on `http://localhost:2024` by default. The API is at the root; the bundled frontend is mounted at `/app`. Open `http://localhost:2024/app` in your browser — you should see a chat interface to the tutor that comes built-in with Deep Agents.

Send a message and watch the agent load the skill file and begin the lesson.

## Step 4 — Deploy to Production

When the agent behaves the way you want locally, deploy it:

```bash
uv run deepagents deploy
```

The CLI builds a Docker image, uploads it, and creates a new deployment revision. When it finishes, it prints the deployment URL.

Open the URL — your agent is now running in production, accessible from anywhere.

<Tip>

**Anonymous auth means the endpoint is open.** Anyone with the URL can send messages. This is fine for demos and internal tools. For a real product you would set `provider = "supabase"` or another 3rd party auth provider in `[auth]` to require user authentication.

</Tip>

## What You Built

A deployed agent from four files:

| File | What it did |
|---|---|
| `deepagents.toml` | Named the agent, set the model, enabled the UI |
| `AGENTS.md` | Gave the agent its instructions |
| `mcp.json` | Connected it to the LangChain docs |
| `skills/module-1/` | Loaded structured teaching content on demand |

No Python, no `langgraph.json`, no Dockerfile. The CLI handled all of that from the config files.

---

## Recap

- `deepagents dev` bundles the project and runs a local server — same API as production, no deploy needed for iteration.
- `deepagents deploy` builds an image, uploads secrets from `.env`, and creates a production revision on LangSmith.
- The CLI generates `langgraph.json`, `deploy_graph.py`, `auth.py`, and `pyproject.toml` from your config files — no manual wiring required.
- The same LangGraph deployment infrastructure from Module 1 (containers, Postgres, Redis, checkpointer, store) runs underneath — the CLI just configures it for you.
