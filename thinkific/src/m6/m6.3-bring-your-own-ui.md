# Bring Your Own UI

In the previous lesson you used the CLI's built-in React UI. That UI is convenient, but you may want to use your own — say, the same `deep_tutor` UI you've been using throughout this course. This lesson shows that the same UI works against a CLI-deployed agent with minimal changes.

The point of this lesson is **API compatibility**, not web development. The CLI deploys a standard LangSmith deployment. The same SDK calls (`client.threads.create()`, `client.runs.stream(...)`, etc.) work against any deployment. So a UI built for one deployment runs against another with just a URL change.

---

## What we'll do

`deep_tutor`'s UI lives at `python/deep_tutor/ui/index.html` and is served by `python/deep_tutor/ui/app.py`. That `app.py` runs **inside** the deep_tutor deployment as a custom route (see m2.3 "Custom Routes").

For a CLI deployment we can't host custom routes — the CLI is opinionated and only ships the standard Agent Server endpoints. So we run a small local UI server on your machine that:

- Serves `deep_tutor`'s `index.html` (the exact same file — no HTML changes)
- Translates UI requests to LangGraph SDK calls against the CLI deployment

This local server lives at `python/m6/ui/app.py`.

---

## A note on auth

This local UI server stubs `/register` and `/login` so they accept any input to simplify this example. The interface remains the same and could be extended to use an auth provider such as Supabase.

---

## Steps

### 1. Get your CLI deployment URL

From m6.2 you deployed `dacli_tutor` via `deepagents deploy`. Copy that deployment URL.

### 2. Start the local UI server

```bash
cd lca-deployment/python/m6/ui
uv run python app.py <your CLI deployment URL>
```

The server starts on `http://localhost:8000` and connects to your CLI deployment for chat traffic.

### 3. Open the UI

```bash
open http://localhost:8000
```

You'll see `deep_tutor`'s familiar login/register page. Type any email — auth is stubbed — and you'll land on the module-1 chat. Send a message. The response streams from your CLI-deployed agent.

### 4. Shut Down the Deployment

When you are done, delete the deployment to avoid unnecessary charges. First list your deployments to get the ID:

```bash
uv run langgraph deploy list
```

Copy the deployment ID from the output, then delete it:

```bash
uv run langgraph deploy delete <DEPLOYMENT_ID>
```

---

## Updates for deepagents CLI

- The majority of the app code remains the same because the endpoints are the same.
- The deepagents CLI organization makes it more convenient to have a single assistant rather than an assistant per module as in the original.
- The per-user management in the deepagents CLI is integral with auth, while in the original `deep_tutor` it was independent.

---

## Recap

- A CLI deployment is a LangSmith deployment. Existing client code works against it.
- The deepagents CLI doesn't host custom HTTP routes, so any UI that depends on custom routes (like `deep_tutor`'s) runs as a separate local server that talks to the deployment via the SDK.
