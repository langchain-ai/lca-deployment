# Using the Tutor

The tutor UI is served directly from your deployment — no local server to run. Open your deployment URL in a browser and you'll land on the chat interface.

For local development, start the agent with `langgraph dev` and open `http://127.0.0.1:2024`.

---

## The Files

```
ui/
  app.py / app.ts    ← custom routes app; serves the UI and proxies requests to the agent
  agent_client.py    ← all LangGraph SDK calls
  index.html         ← single-page chat UI
```

The custom routes app holds your `LANGSMITH_API_KEY` and makes all LangGraph SDK calls. The browser talks only to the app — the API key is never exposed to the client.

---

## Configure and Start

You will land on the **Config** tab.

1. **First name / Last name** — used to personalize the agent and create your student namespace in the Store.
2. **Goals / Preferences** — optional, stored in the LangGraph Store alongside your profile.
3. Click **Start**.

Clicking Start creates one assistant and one thread per lesson and saves them in your browser's local storage. These persist across page reloads — clicking Start again creates a fresh set.

---

## Start Chatting

Switch to the **Lesson 1** tab. The agent is ready.

Try asking:

- *"What is the Agent Server?"*
- *"Walk me through how a run flows through the system."*
- *"Quiz me on what we've covered."*

The agent will read from `tutor_l1_instructions.md` to guide the lesson and call the `read_lesson_material` tool when it needs to answer a specific question.

---

## Recap

- The UI is served directly from the deployment — no local server needed.
- **Start** creates one assistant + thread per lesson and stores them in localStorage. Clicking it again resets your sessions.
- The agent's behavior is driven by the context set on each assistant — `lesson_id`, `student_name`, `store_namespace`.
