# Using the Tutor

The tutor UI is a simple local web app. It is **not** part of the LangSmith deployment — it runs on your machine and talks to the deployment over the LangGraph SDK.

---

## The Files

```
ui/
  server.py / server.ts   ← local server; proxies requests to LangSmith
  index.html              ← single-page chat UI
```

The server holds your `LANGSMITH_API_KEY` and `LANGGRAPH_URL` and makes all LangGraph SDK calls. The browser talks only to the local server.

The server exposes four endpoints:

| Endpoint | What it does |
|---|---|
| `GET /lessons` | Returns the list of lessons and which are active |
| `POST /student/start` | Creates one assistant + one thread per lesson; writes the student profile to the Store |
| `POST /student/update` | Updates the student's goals and preferences in the Store |
| `POST /chat` | Streams a response from the agent for a given assistant + thread |

## Configure and Start

You will land on the **Config** tab.

1. **Deployment URL** — paste the URL from the LangSmith Deployments tab (from lesson M1.2). For local dev use `http://127.0.0.1:2024`.
2. **First name / Last name** — used to personalize the agent and create your student namespace in the Store.
3. **Goals / Preferences** — optional, stored in the LangGraph Store alongside your profile.
4. Click **Start**.

Clicking Start creates one assistant and one thread per lesson and saves them in your browser's local storage. These persist across page reloads — clicking Start again creates a fresh set.

---

## Start Chatting

Switch to the **Lesson 1** tab. The agent is ready.

Try asking:

- *"What is the Agent Server?"*
- *"Walk me through how a run flows through the system."*
- *"Quiz me on what we've covered."*

The agent will read from `tutor_l1_instructions.md` to guide the lesson and call the `read_lesson_material` tool when it needs to answer a specific question.

---

## Recap

- The server proxies all LangGraph SDK calls to the deployment.
- **Start** creates one assistant + thread per lesson and stores them in localStorage. Clicking it again resets your sessions.
- The agent's behavior is driven by the context set on each assistant — `lesson_id`, `student_name`, `store_namespace` — exactly as covered in lesson M1.1.
