# Using the Tutor

The tutor agent comes with a browser-based chat UI for registering as a student, selecting a module, and chatting with the agent. It is served directly from the deployment alongside the agent, so there is no separate server to run.

In the previous lesson you deployed the agent to LangSmith. Open your deployment URL in a browser to access it.

If you have not deployed yet and want to follow along locally, run `langgraph dev` and open `http://127.0.0.1:2024` instead.

`app.py` / `app.ts` runs server-side and proxies all LangGraph SDK calls. The browser talks only to it — never directly to the agent API.

## Register

You will land on the **setup** tab with a login/register form. On your first visit, click **register** and fill in:

1. **First name / Last name** — stored in the LangGraph Store and used to personalize the agent.
2. **Email** — used to derive your namespace in the Store (e.g. `jane@example_com` — dots replaced by underscores).
3. **Course goals** — optional, stored alongside your profile.

Clicking **register** creates one assistant and one thread per module and saves them in your browser's `localStorage`. These persist across page reloads. On return visits, use **log in** with your email to restore your sessions.

---

## Start Chatting

After registering or logging in, the module tabs appear. Click **deployment architecture** to open the chat for this module.

Try asking:

- *"What is the Agent Server?"*
- *"Walk me through how a run flows through the system."*
- *"Quiz me on what we've covered."*

On its first response the agent reads `/skills/module-1/SKILL.md`, which contains the teaching plan for this module. When it needs to answer a factual question it reads `/skills/module-1/information.md`.

---

## Recap

- The UI is served from the deployment — no separate server needed.
- **Register** creates one assistant + thread per module and stores them in `localStorage`. **Log in** restores an existing session by email.
- Each assistant is configured with `module_id` and `store_namespace`. The agent reads the student profile from the Store on every run using the namespace.
- The agent's teaching behavior is driven by the skill files in `/skills/` — not hardcoded in the agent.
