LangSmith Deployment is a workflow orchestration runtime purpose-built for agent workloads. It provides the managed infrastructure agents need to run reliably in production at scale, supporting the full lifecycle from local development to deployment.

Examining the components of a deployment can clarify what this means in practice. The diagram below takes you through the steps and packages that combine to build a deployment.

## Layers

<div style="overflow:hidden;margin:1.5rem 0;">
<img src="images/langgraph_stack.svg" alt="LangGraph deployment stack" style="float:left;width:48%;margin-right:2rem;"/>
<div style="font-size:0.95rem;line-height:1.7;">
<p><strong>Infrastructure</strong> — the managed data layer underneath everything: a Postgres database (threads, assistants, cron jobs, store, checkpointer) and Redis for the run queue. LangSmith provisions and operates this for you.</p>
<p><strong>langgraph-api</strong> — the proprietary server that sits on top of infrastructure. It includes the API server (HTTP endpoints), workers (processes that execute runs), a queue adapter (connecting workers to Redis), checkpointer and store adapters (connecting to Postgres), and auth middleware.</p>
<p><strong>langgraph (OSS)</strong> — the open-source execution engine. It compiles your <code>StateGraph</code> and drives the Pregel loop that steps through your nodes. <code>langgraph-api</code> imports and runs it.</p>
<p><strong>Your code</strong> — your <code>agent.py</code> (or <code>agent.ts</code>). You define the graph; LangGraph compiles and executes it; <code>langgraph-api</code> hosts it.</p>
</div>

# Course Format
The course is broken into modules. Modules focus on aspects of deployment. The modules will generally follow three steps:

1) Introduce a concept
2) Use the concept in simple examples
3) Use the concept in a larger example: A tutor agent.

The course text can generally be used stand-alone. The video will summarize the content, run examples, and work in the UI. The UI content will generally not be in the text.

LangSmith Deployment is evolving along with agents - rapidly! The course may differ from the UI over time, and new modules may be added as new features are added.

**Use a Coding Agent** — Throughout the course, you will see snippets of code that are meant to show a realization of a concept. Using a coding agent to build your agents and deployment package is encouraged. The Getting Set Up section describes how to include the LangChain Docs MCP server to aid your agent.

## Shutting Down Deployments

Each deployment you create incurs charges while it is running. When you are done with a deployment, delete it. First list your deployments to get the ID:

```bash
uv run langgraph deploy list
```

Then delete by ID:

```bash
uv run langgraph deploy delete <DEPLOYMENT_ID>
```
